# Incident Response Runbook

## Trigger Events
- private vulnerability report received
- critical dependency vulnerability detected
- release provenance verification failure

## Steps
1. Intake and acknowledge report.
2. Triage severity and exploitability.
3. Reproduce in controlled environment.
4. Define containment and fix plan.
5. Prepare patch and tests.
6. Coordinate disclosure and publish advisory.
7. Post-incident review and evidence archival.

## Evidence Requirements
- timeline log
- affected versions
- fix commit references
- verification commands and outputs
